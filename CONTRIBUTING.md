# Contributing to Sharemovies

Thank you for your interest in contributing to Sharemovies! We welcome contributions from the community to help make this project even better.

Before you start, please take a moment to review this document. It provides important information to guide you through the contribution process.

## Getting Started

- Make sure you have a GitHub account.
- Fork the repository on GitHub.

## Making Contributions

1. Create a new branch on your fork with a descriptive name for your feature or bug fix.

   ```bash
   git checkout -b feature/my-new-feature
   ```

   Please use a branch name that clearly describes the changes you're making.

2. Make your changes and commit them to your branch.

3. If you've added code that should be tested, add tests.

4. Ensure your code follows our coding standards and conventions.

5. Make sure your commits are descriptive and follow our [commit message guidelines](CONTRIBUTING.md#commit-message-guidelines).

6. Push your changes to your fork on GitHub.

7. Create a pull request (PR) from your branch to the main repository's `main` branch.



## Commit Message Guidelines

- We follow conventional commit messages. Please format your commit messages as described in [CONTRIBUTING.md](CONTRIBUTING.md#commit-message-guidelines).
